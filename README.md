## このリポジトリについて

社会人向け勉強コミュニティで主催した **AI入門オフ会（2026年・関西）** の登壇資料です。

## 活動の背景・取り組み

### 課題の発見

社会人向け勉強コミュニティのIT支援企画に携わる中で、同じ質問が繰り返し寄せられることに気づきました。  
原因を掘り下げると、本質的なハードルは **「ツールの難しさ」ではなく、最初の一歩を踏み出す心理的障壁** にあると分かりました。

### 設計の工夫

- **インストール〜活用事例まで**を一連の流れとして設計し、「とりあえず動かせた」体験を最初に提供
- **自分一人が教える構造を避け**、参加者同士が質問し合える場づくりを意識
- オフ会後も学びが続くよう、記事の執筆など **自走できる仕組み** を整備

### 結果と学び

オフ会を通じて、実際にAIを日常業務に活用し始めた参加者が生まれました。  
この経験から次の2点を学びました。

1. **相手の言葉にならない困りごとを汲み取ること** — 「分からない」と言えない空気を壊し、本音を引き出す関係づくりの大切さ
2. **「使い続けられる状態」まで設計すること** — 一過性のイベントで終わらせず、その後の行動変容まで見据えた構造設計

この姿勢を、顧客と長く伴走するシステム開発の現場でも大切にしていきたいと考えています。
